# UI Components Documentation

## Обзор

Все UI компоненты созданы в минималистичном стиле с бело-синей цветовой схемой, тематика - спорт.

## Цветовая схема

- **Основной синий**: `#2563eb` (primary-blue)
- **Темный синий**: `#1d4ed8` (primary-blue-dark)
- **Светлый синий**: `#3b82f6` (primary-blue-light)
- **Акцентный синий**: `#0ea5e9` (accent-blue)
- **Белый фон**: `#ffffff`
- **Светлый фон**: `#f8fafc`
- **Светло-синий фон**: `#eff6ff`

## Компоненты

### 1. Регистрация (`/register`)
**Файл**: `src/components/RegistrationForm.tsx`

Форма регистрации нового пользователя:
- Поле логина
- Поле пароля
- Подтверждение пароля
- Валидация (минимум 6 символов)
- Переход на страницу входа после успешной регистрации

**API**: `POST /api/auth/register`

### 2. Авторизация (`/login`)
**Файл**: `src/components/LoginForm.tsx`

Форма входа в систему:
- Поле логина
- Поле пароля
- Сохранение токена в localStorage
- Переход в профиль после успешного входа

**API**: `POST /api/auth/login`

**Примечание**: Форма Spring Security отключена, используется кастомная авторизация через API.

### 3. Профиль участника (`/profile`)
**Файл**: `src/components/ParticipantProfile.tsx`

Личный профиль пользователя:
- Имя участника
- Название команды (если есть)
- Список всех активностей участника с фото
- Кнопки навигации:
  - Добавить активность
  - Создать команду
  - Трекер команд
  - Выйти

**API**: `GET /api/participants/{userId}`

### 4. Добавить команду (`/add-team`)
**Файл**: `src/components/AddTeamForm.tsx`

Форма создания новой команды:
- Название команды
- Поиск участников по имени
- Добавление участников в команду
- Список выбранных участников с возможностью удаления

**API**: 
- `GET /api/participants/search?query={query}` - поиск участников
- `POST /api/teams` - создание команды

### 5. Добавить активность (`/add-activity`)
**Файл**: `src/components/AddActivityForm.tsx`

Форма добавления спортивной активности:
- Выбор команды (dropdown)
- Тип активности (текстовое поле)
- Энергия/баллы (числовое поле)
- Загрузка фото (опционально)
- Предпросмотр загруженного фото

**API**: 
- `GET /api/teams` - получение списка команд
- `POST /api/activities` - создание активности (multipart/form-data)

### 6. Профиль команды (`/team/:teamId`)
**Файл**: `src/components/TeamProfile.tsx`

Страница команды с полной информацией:
- Название команды
- Количество участников
- Общее количество баллов
- Карточки всех активностей команды с:
  - Фото (если есть)
  - Тип активности
  - Имя участника
  - Баллы
  - Дата

**API**: `GET /api/teams/{teamId}`

### 7. Трекер команд (`/tracker`)
**Файл**: `src/components/TeamTracker.tsx`

Сравнительный рейтинг всех команд:
- Список команд отсортированный по баллам
- Медали для топ-3 команд (золото, серебро, бронза)
- Для каждой команды:
  - Место в рейтинге
  - Название
  - Количество участников
  - Общие баллы
- Статистика:
  - Всего команд
  - Лучший результат
  - Средний балл

**API**: `GET /api/teams/rankings`

## Базовые UI компоненты

### Button (`src/components/ui/Button.tsx`)
Кнопка с тремя вариантами:
- `primary` - основная синяя кнопка
- `outline` - кнопка с обводкой
- `ghost` - прозрачная кнопка

### Input (`src/components/ui/Input.tsx`)
Текстовое поле с label и стилизацией

## Маршрутизация

Все маршруты защищены компонентом `PrivateRoute`, который проверяет наличие токена в localStorage.

### Публичные маршруты:
- `/` - редирект на `/login`
- `/register` - регистрация
- `/login` - вход

### Защищенные маршруты:
- `/profile` - профиль участника
- `/add-team` - создание команды
- `/add-activity` - добавление активности
- `/team/:teamId` - профиль команды
- `/tracker` - трекер команд

## Иконки

Используется библиотека `lucide-react` для всех иконок:
- `User` - пользователь
- `Users` - команда
- `Activity` - активность
- `Trophy` - трофей/рейтинг
- `LogIn/LogOut` - вход/выход
- `Search` - поиск
- `Upload` - загрузка
- `Medal` - медаль
- И другие

## Адаптивность

Все компоненты адаптивны и корректно отображаются на:
- Мобильных устройствах (< 768px)
- Планшетах (768px - 1024px)
- Десктопах (> 1024px)

## Запуск

```bash
cd frontend
npm install
npm start
```

Приложение запустится на `http://localhost:3000`
Backend должен быть запущен на `http://localhost:8082` (настроено в package.json proxy)
